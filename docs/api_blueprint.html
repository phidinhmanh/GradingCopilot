Hợp đồng API (API Blueprint Specification) - v1

Tài liệu này định nghĩa chi tiết các API endpoints của hệ thống AI Grading Copilot, đóng vai trò là Hợp đồng kỹ thuật (Contract) giữa Frontend (Next.js) và Backend (FastAPI).

1. Quy chuẩn chung (Global Conventions)

  • Base URL: /api/v1
  • Content-Type: application/json
  • Authentication: JWT Bearer Token gửi qua header: Authorization: Bearer <token> (hoặc Cookie bảo mật).
  • Định dạng lỗi (Error Response): json { "detail": "Mô tả chi tiết lỗi xảy ra" }

MÔ-ĐUN 1: XÁC THỰC (AUTHENTICATION)

1. Đăng ký tài khoản

  • Endpoint: POST /auth/register
  • Mô tả: Đăng ký tài khoản cho Giáo viên hoặc Học sinh.
  • Payload: json { "email": "student1@school.edu.vn", "password": "SecurePassword123", "full_name": "Nguyễn Văn A", "role": "student" // Hoặc "teacher" }
  • Phản hồi thành công (201 Created): json { "id": "e838efca-8178-433a-bd1e-53c072d6279f", "email": "student1@school.edu.vn", "full_name": "Nguyễn Văn A", "role": "student", "created_at": "2026-06-05T08:00:00Z" }

2. Đăng nhập

  • Endpoint: POST /auth/login
  • Payload: json { "email": "student1@school.edu.vn", "password": "SecurePassword123" }
  • Phản hồi thành công (200 OK): json { "access_token": "eyJhbGciOi...", "token_type": "bearer", "user": { "id": "e838efca-8178-433a-bd1e-53c072d6279f", "email": "student1@school.edu.vn", "full_name": "Nguyễn Văn A", "role": "student" } }

MÔ-ĐUN 2: QUẢN LÝ LỚP HỌC (CLASSROOMS)

1. Tạo lớp học mới (Chỉ dành cho Giáo viên)

  • Endpoint: POST /classrooms
  • Headers: Authorization: Bearer <teacher_jwt>
  • Payload: json { "name": "Lớp 8A1 - Môn Văn" }
  • Phản hồi thành công (201 Created): json { "id": "a90df12d-118c-4a30-9b48-1212abfde211", "name": "Lớp 8A1 - Môn Văn", "class_code": "VVAN8A1", // Sinh ngẫu nhiên và duy nhất để HS join "teacher_id": "e838efca-8178-433a-bd1e-53c072d6279f" }

2. Lấy danh sách lớp học

  • Endpoint: GET /classrooms
  • Headers: Authorization: Bearer <jwt_token>
  • Mô tả: Trả về danh sách lớp học mà người dùng quản lý (nếu là Giáo viên) hoặc đã tham gia (nếu là Học sinh).
  • Phản hồi thành công (200 OK): json [ { "id": "a90df12d-118c-4a30-9b48-1212abfde211", "name": "Lớp 8A1 - Môn Văn", "class_code": "VVAN8A1", "teacher_name": "Nguyễn Minh", "created_at": "2026-06-05T08:00:00Z" } ]

3. Học sinh tham gia lớp học bằng mã (Yêu cầu Đăng nhập)

  • Endpoint: POST /classrooms/join
  • Headers: Authorization: Bearer <student_jwt>
  • Mô tả: Học sinh tham gia lớp học bằng cách nhập mã lớp. Hệ thống sẽ tự động liên kết user_id từ JWT token và tạo bản ghi trong classroom_students.
  • Payload: json { "class_code": "VVAN8A1", "alias_name": "Nguyễn Văn A" // Không bắt buộc, dùng để ghi đè tên hiển thị trong lớp }
  • Phản hồi thành công (201 Created): json { "id": "b1827cfd-11bb-4554-aa22-544bbddcc77b", // ID của bản ghi tham gia (classroom_students.id) "classroom_id": "a90df12d-118c-4a30-9b48-1212abfde211", "classroom_name": "Lớp 8A1 - Môn Văn" }

4. Lấy danh sách học sinh trong lớp (Chỉ dành cho Giáo viên)

  • Endpoint: GET /classrooms/{classroom_id}/students
  • Headers: Authorization: Bearer <teacher_jwt>
  • Mô tả: Trả về danh sách học sinh đã tham gia lớp học.
  • Phản hồi thành công (200 OK): json [ { "user_id": "e838efca-8178-433a-bd1e-53c072d6279f", "email": "student1@school.edu.vn", "full_name": "Nguyễn Văn A", "alias_name": "A Văn Nguyễn", "joined_at": "2026-06-05T08:20:00Z" } ]

5. Xóa học sinh khỏi lớp (Chỉ dành cho Giáo viên)

  • Endpoint: DELETE /classrooms/{classroom_id}/students/{user_id}
  • Headers: Authorization: Bearer <teacher_jwt>
  • Mô tả: Xóa một học sinh ra khỏi lớp học. Khi học sinh bị xóa, các bài nộp của học sinh này trong các bài tập thuộc lớp này cũng có thể bị ảnh hưởng (tùy thuộc vào chính sách xóa của cơ sở dữ liệu - Cascade delete).
  • Phản hồi thành công (204 No Content): Không trả về dữ liệu.

MÔ-ĐUN 3: QUẢN LÝ TIÊU CHÍ CHẤM ĐIỂM (RUBRICS)

1. Tạo Rubric mới (Chỉ dành cho Giáo viên)

  • Endpoint: POST /rubrics
  • Headers: Authorization: Bearer <teacher_jwt>
  • Mô tả: Tạo một bộ Rubric tiêu chí chấm điểm mới để có thể sử dụng lại cho nhiều bài tập khác nhau.
  • Payload: json { "title": "Rubric NLXH 200 chữ", "subject": "Ngữ văn", "criteria": [ { "id": "c1", "name": "Bố cục & Lập luận", "max_points": 5.0, "description": "Mở đoạn rõ ràng, dẫn chứng thuyết phục." }, { "id": "c2", "name": "Ngữ pháp & Chính tả", "max_points": 5.0, "description": "Không sai quá 2 lỗi chính tả/ngữ pháp." } ] }
  • Phản hồi thành công (201 Created): json { "id": "g90df77a-abcd-1234-5678-999988887777", "title": "Rubric NLXH 200 chữ", "subject": "Ngữ văn", "criteria": [ ... ], "created_by": "teacher-uuid", "created_at": "2026-06-05T08:15:00Z" }

2. Lấy danh sách Rubric của Giáo viên

  • Endpoint: GET /rubrics
  • Headers: Authorization: Bearer <teacher_jwt>
  • Phản hồi thành công (200 OK): json [ { "id": "g90df77a-abcd-1234-5678-999988887777", "title": "Rubric NLXH 200 chữ", "subject": "Ngữ văn", "created_at": "2026-06-05T08:15:00Z" } ]

3. Lấy chi tiết Rubric

  • Endpoint: GET /rubrics/{id}
  • Headers: Authorization: Bearer <jwt_token>
  • Phản hồi thành công (200 OK): json { "id": "g90df77a-abcd-1234-5678-999988887777", "title": "Rubric NLXH 200 chữ", "subject": "Ngữ văn", "criteria": [ ... ], "created_at": "2026-06-05T08:15:00Z" }

MÔ-ĐUN 4: BÀI TẬP VỀ NHÀ (ASSIGNMENTS)

1. Tạo bài tập về nhà mới (Chỉ dành cho Giáo viên)

  • Endpoint: POST /assignments
  • Headers: Authorization: Bearer <teacher_jwt>
  • Mô tả: Tạo bài tập mới. Giáo viên có thể truyền rubric_id của một Rubric đã tạo sẵn HOẶC truyền cấu trúc rubric (inline) để tự động tạo một Rubric mới gắn liền với bài tập này.
  • Payload: json { "title": "Bài viết Nghị luận xã hội số 1", "description": "Nêu suy nghĩ của em về lòng dũng cảm (viết đoạn văn 200 chữ).", "deadline": "2026-06-12T23:59:00Z", "classroom_id": "a90df12d-118c-4a30-9b48-1212abfde211", "answer_key_url": "https://r2.gradingapp.com/keys/dapan_bai1.txt", // Tùy chọn "rubric_id": "g90df77a-abcd-1234-5678-999988887777", // Tùy chọn: Dùng rubric có sẵn "rubric": { // Tùy chọn: Tạo rubric inline nếu không truyền rubric_id "title": "Rubric NLXH 200 chữ", "criteria": [ { "id": "c1", "name": "Bố cục & Lập luận", "max_points": 5.0, "description": "Mở đoạn rõ ràng, dẫn chứng thuyết phục." } ] } }
  • Phản hồi thành công (201 Created): json { "id": "c567feab-129d-4001-ba81-1234abcdffee", "title": "Bài viết Nghị luận xã hội số 1", "classroom_id": "a90df12d-118c-4a30-9b48-1212abfde211", "rubric_id": "g90df77a-abcd-1234-5678-999988887777", "created_at": "2026-06-05T08:15:00Z" }

2. Lấy danh sách bài tập của một lớp

  • Endpoint: GET /assignments/class/{classroom_id}
  • Headers: Authorization: Bearer <jwt_token>
  • Mô tả: Lấy danh sách các bài tập được giao trong một lớp học cụ thể.
  • Phản hồi thành công (200 OK): json [ { "id": "c567feab-129d-4001-ba81-1234abcdffee", "title": "Bài viết Nghị luận xã hội số 1", "description": "Nêu suy nghĩ của em về lòng dũng cảm (viết đoạn văn 200 chữ).", "deadline": "2026-06-12T23:59:00Z", "created_at": "2026-06-05T08:15:00Z" } ]

3. Lấy danh sách bài nộp của bài tập (Chỉ dành cho Giáo viên)

  • Endpoint: GET /assignments/{assignment_id}/submissions
  • Headers: Authorization: Bearer <teacher_jwt>
  • Mô tả: Giáo viên lấy danh sách bài nộp của cả lớp cho một Assignment cụ thể để theo dõi ai đã nộp, ai chưa nộp, điểm đề xuất của AI và điểm chốt cuối cùng.
  • Phản hồi thành công (200 OK): json [ { "id": "f892ccdd-1234-abcd-5678-eeeeffffbbbb", "student_id": "e838efca-8178-433a-bd1e-53c072d6279f", "student_name": "Nguyễn Văn A", "status": "graded", // processing | graded | error "submitted_at": "2026-06-05T08:20:00Z", "grade_summary": { "proposed_score": 7.5, "final_score": 8.0, // NULL nếu giáo viên chưa duyệt "is_approved": true } } ]

MÔ-ĐUN 5: BÀI NỘP HỌC SINH & DUYỆT ĐIỂM (SUBMISSIONS & GRADING)

1. Yêu cầu sinh Presigned URL upload ảnh (Chỉ dành cho Học sinh)

  • Endpoint: POST /submissions/presign
  • Headers: Authorization: Bearer <student_jwt>
  • Payload: json { "file_name": "homework_page1.jpg", "content_type": "image/jpeg" }
  • Phản hồi thành công (200 OK): json { "upload_url": "https://grading-copilot.r2.cloudflare.com/temp/uuid-file.jpg?X-Amz-Signature=...", "download_url": "https://grading-copilot.r2.cloudflare.com/temp/uuid-file.jpg" }

1.5 Trích xuất văn bản OCR từ ảnh (Chỉ dành cho Học sinh)

  • Endpoint: POST /submissions/ocr
  • Headers: Authorization: Bearer <student_jwt>
  • Mô tả: Nhận ảnh bài tập vừa upload và thực hiện chạy OCR nhanh (1-3s) để trích xuất văn bản thô. Text này sẽ được trả về trực tiếp để học sinh có thể kiểm tra và tự sửa lỗi/mờ trên UI trước khi chính thức bấm nộp bài chấm AI. Tránh việc chấm nhầm ảnh mờ gây lãng phí tài nguyên LLM.
  • Payload: json { "image_url": "https://grading-copilot.r2.cloudflare.com/temp/uuid-file.jpg" }
  • Phản hồi thành công (200 OK): json { "raw_text": "Hôm nay em xin viết về lòng dũng cảm..." }

2. Nộp bài tập (Bảo mật - Định danh từ JWT)

  • Endpoint: POST /submissions
  • Headers: Authorization: Bearer <student_jwt>
  • Mô tả: Nộp bài làm kèm văn bản đã được kiểm tra/chỉnh sửa bởi học sinh.
    [!NOTE] > Cơ chế nộp lại (Resubmission): Nếu học sinh gọi endpoint này khi đã nộp bài tập trước đó, API sẽ tự động xử lý ghi đè (overwrite) nếu trạng thái bài nộp cũ chưa được duyệt (is_approved = false). Do FastAPI BackgroundTasks / webhook queue trên Render không hỗ trợ hủy thread chạy ngầm một cách native, hệ thống áp dụng cơ chế Checkpoint Verification (Hủy thụ động): Khi ghi đè, bản ghi DB cũ bị xóa/thay thế bằng ID mới. Task cũ chạy ngầm khi đến checkpoint kiểm tra DB sẽ thấy ID cũ không tồn tại và tự động dừng lại (abort) trước khi gọi LLM. Nếu bài đã được duyệt điểm, hệ thống trả về 409 Conflict.
  • Ràng buộc kiểm tra của API:
  • Kiểm tra xem học sinh (user_id) đã join lớp của bài tập (assignment_id) chưa. Nếu chưa, trả về 403 Forbidden.
  • Kiểm tra xem bài tập này đã được giáo viên duyệt điểm chưa. Nếu đã duyệt rồi, trả về 409 Conflict.
  • Payload: json { "assignment_id": "c567feab-129d-4001-ba81-1234abcdffee", "image_url": "https://grading-copilot.r2.cloudflare.com/temp/uuid-file.jpg", "confirmed_text": "Văn bản bài làm học sinh đã kiểm tra hoặc chỉnh sửa..." }
  • Phản hồi thành công (202 Accepted): json { "id": "f892ccdd-1234-abcd-5678-eeeeffffbbbb", "status": "processing", // processing | graded | error "message": "Nộp bài thành công. Tiến trình chấm điểm tự động đã bắt đầu." }

3. Hủy / Thu hồi bài nộp (Chỉ dành cho Học sinh)

  • Endpoint: DELETE /submissions/{id}
  • Headers: Authorization: Bearer <student_jwt>
  • Mô tả: Học sinh có thể chủ động hủy/rút lại bài làm đã nộp nếu bài nộp chưa được giáo viên duyệt điểm (is_approved = false). Khi thu hồi, bản ghi tương ứng và kết quả chấm điểm tạm thời trong bảng grades sẽ bị xóa khỏi cơ sở dữ liệu. Task chạy ngầm chấm bài cũ khi đến checkpoint kiểm tra DB sẽ tự động dừng lại (abort) trước khi thực hiện gọi LLM tốn phí.
  • Phản hồi thành công (204 No Content): Không trả về dữ liệu.

4. Lấy chi tiết bài nộp (Đã tích hợp đầy đủ chi tiết điểm số & nhận xét)

  • Endpoint: GET /submissions/{id}
  • Headers: Authorization: Bearer <jwt_token>
  • Mô tả: Lấy thông tin chi tiết bài nộp bao gồm ảnh gốc và text OCR. Nếu trạng thái là graded, API trả về kèm theo thông tin chi tiết điểm số, tiêu chí và nhận xét (từ AI lẫn Giáo viên) dưới trường grade. Nếu trạng thái là processing hoặc error, trường grade sẽ trả về null.
  • Phản hồi thành công (200 OK): json { "id": "f892ccdd-1234-abcd-5678-eeeeffffbbbb", "assignment_id": "c567feab-129d-4001-ba81-1234abcdffee", "student_name": "Nguyễn Văn A", "image_url": "https://...", "ocr_text": "Hôm nay em xin viết về lòng dũng cảm...", "status": "graded", // processing | graded | error "submitted_at": "2026-06-05T08:20:00Z", "grade": { "id": "g90df77a-abcd-1234-5678-999988887777", "proposed_score": 7.5, // Điểm AI đề xuất "final_score": 8.0, // NULL nếu giáo viên chưa chốt duyệt điểm "is_approved": true, "ai_feedback": "Bài viết trôi chảy, luận điểm dũng cảm tốt. Có 1 lỗi chính tả ở dòng 3.", "final_feedback": "Thầy đồng ý với nhận xét của AI. Bài viết rất có cảm xúc, cố gắng khắc phục lỗi chính tả nhé em!", "grading_details": { "c1": { "score": 4.5, "max_points": 5.0, "reason": "Thầy cộng thêm 0.5 điểm khuyến khích vì ý tưởng sáng tạo." }, "c2": { "score": 3.5, "max_points": 5.0, "reason": "Giữ nguyên lỗi chính tả." } }, "graded_at": "2026-06-05T08:30:00Z" } }

5. Phê duyệt hoặc sửa điểm (Chỉ dành cho Giáo viên)

  • Endpoint: PUT /submissions/{id}/approve
  • Headers: Authorization: Bearer <teacher_jwt>
  • Payload: json { "final_score": 8.0, // Giáo viên sửa lại điểm cao hơn hoặc giữ nguyên "final_feedback": "Thầy đồng ý với nhận xét của AI. Bài viết rất có cảm xúc, cố gắng khắc phục lỗi chính tả nhé em!", "grading_details": { "c1": { "score": 4.5, "reason": "Thầy cộng thêm 0.5 điểm khuyến khích vì ý tưởng sáng tạo." }, "c2": { "score": 3.5, "reason": "Giữ nguyên lỗi chính tả." } } }
  • Phản hồi thành công (200 OK): json { "submission_id": "f892ccdd-1234-abcd-5678-eeeeffffbbbb", "final_score": 8.0, "is_approved": true, "approved_at": "2026-06-05T08:30:00Z" }
